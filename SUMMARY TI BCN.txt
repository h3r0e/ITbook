-CREACIÓN DE USUARIOS Y GESTIÓN DE RECURSOS
	WINDOWS SERVER: 1-) CREACIÓN USUARIOS ATRAVÉS DEL SERVIDOR WINDOWS (ACTIVE DIRECTORY)
		        2-) PARA NO TENER QUE ACCEDER A LA MÁQUINA DESCARGAMOS HERRAMIENTAS ADMINISTRATIVAS.
			3-) HABILITAMOS SERVICIO DNS Y DHCP PARA ADMINISTRACIÓN DE SERVIDORES Y WORKSTATIONS.
			4-) ASIGNACIÓN DE UN USUARIO A UNA UNIDAD ORGANIZATIVA.		        
			
			GPO:
			1-) MAPEAR CARPETAS COMPARTIDAS.
			2-) COMPARTIR IMPRESORAS.
			3-) ACTUALIZACIONES DE WINDOWS PROGRAMADAS.
			4-) HABILITACIÓN DE ESCRITORIO REMOTO (SOLO ADMINS), EN USUARIOS NORMALES DAMOS ACCESO A LA
			    MÁQUINA QUE VAN A USAR ÚNICAMENTE.
			5-) ACCESO A RECURSOS DE ADMINISTRACIÓN DEL SISTEMA: CMD, POWERSHELL, ETC.
			6-) CORRER SCRIPTS, INSTALACIÓN AUTOMATIZADA DE PROGRAMAS, ETC. 
			7-) UNIDADES ORGANIZATIVAS QUE CONTIENEN GPOs CONCRETAS, LA INFORMACIÓN SE GUARDA Y SE RESERVA LO MÁXIMO POSIBLE.

-COPIAS DE SEGURIDAD
	NAS (SYNOLOGY): 1-) SE MONTA Y SE CONECTA CABLE ETHERNET PARA ACCEDER POR IP
			2-) NOS DIRIGIMOS A ADMINISTRADOR DE ALMACENAMIENTO
			3-) CREAMOS RAID. EN ESTE CASO RAID 6 --> DATOS DE PARIDAD EN 2 DISCOS, REQUIERE ALMENOS 4 DISCOS
							      --> TIENE CAPACIDAD DE RECUPERAR DATOS AUNQUE MUERAN 2 DISCOS.
			4-) UNA VEZ CREADO EL SISTEMA RAID QUE SE VA A UTILIZAR, NOS DIRIGIMOS A HYPER BACKUP
			    SELECCIONAMOS CARPETAS O ARCHIVOS Y CREAMOS UNA TAREA PROGRAMADA PARA QUE REALICE COPIAS DE SEGURIDAD
			    CADA CIERTO TIEMPO (CADA VIERNES, CADA SEMANA, CADA MES, ETC).
			5-) CONFIGURACIÓN ATRAVÉS DE LA INTERFAZ QUE HAY POR IP.

-CREACIÓN Y ASIGNACIÓN DE RECURSOS
	NAS & WINDOWS SERVER: 1-) EL PRIMER PASO ES CONECTAR EL NAS CON WINDOWS SERVER (UNIR NAS AL DOMINIO AD).
			      2-) EN PANEL DE CONTROL >> DOMAIN SE ESCRIBE EL DOMINIO Y LA IP DEL SRVIDOR WINDOWS,
				  LUEGO SE ESPECIFICA EL TIPO DE DOMINO (EN ESTE CASO AD) Y EN MANAGEMENT MODE ESCOGEMOS
				  TRUSTED DOMAIN. POR ÚLTIMO APLICAMOS.
			      3-) UNA VEZ QUE HEMOS UNIDO NAS CON EL SERVICIO DE AD, CREAMOS LAS CARPETAS EN EL NAS,
				  DESPUES LAS COMPARTIMOS ASIGNANDO PERMISOS A LOS GRUPOS O USUARIOS DE NUESTRO AD
				  QUE QUERAMOS Y ATRAVÉS DE LAS GPO DE WINDOWS SERVER, MAPEAMOS LAS UNIDADES DE RED EN SU USUARIO.

-DOCUMENTACIÓN Y GESTIÓN DE INVENTARIO
	OFFICE 365 & ANSIBLE: 1-) DOCUMENTACIÓN DE HARDWARE Y LICENCIAS DEL ESTUDIO EN EXCELS DE OFFICE.
			      2-) USO DE ANSIBLE PARA LISTAR HARDWARE DEL ESTUDIO USANDO COMANDO POWERSHELL EXCEPTO DISCO DURO.
			      3-) CREACIÓN DE PLANTILLA DE COMPRA PARA GESTIONAR PRESUPUESTO Y MATERIAL QUE SE VA A COMPRAR.

-MONITORIZACIÓN DE RECURSOS DE HARDWARE
	ADMINISTRADOR DE TAREAS: 1-) NOS DIRIGIMOS A PERFORMANCE Y VISUALIZAMOS LOS RECURSOS QUE CONSUME LA MÁQUINA.
				 2-) COMPROBAMOS QUE LA GPU NO ESTÉ SOBRECALENTADA (BAJA EL RENDIMIENTO DEL EQUIPO).
				 3-) COMPROBAMOS DATOS DE CPU Y RAM QUE SEAN NORMALES.
				 4-) POR ÚLTIMO VISUALIZAMOS LA VELOCIDAD DE CONEXIÓN DEL EQUIPO. 
				 5-) INSTALACIÓN DE UN PAQUETE DE PROGRAMAS EN UNA MÁQUINA EN CONCRETO (EJEMPLO MÁQUINA XSENSE).

		3-) EN INTERNET HAY INFORMACIÓN DE QUE CONTENIDO TIENE QUE LLEVAR EL FICHERO.

	 	    LA DIRECCIÓN IP DE LAS MÁQUINAS Y SERVIDORES QUE ADMINISTRAREMOS CON EL SERVIDOR.
	
	PLASTIC: SERVIDOR QUE ACTUA COMO DISPARADOR. SE CONFIGURA CREANDO UN REPOSITORIO EN UNA MÁQUINA, POR EJEMPLO LA LOCAL.
		 Y CUANDO VE CAMBIOS, AVISA ATRAVÉS DE SU INTERFAZ SI SE DESEAN SUBIR ESOS CAMBIOS.

-CONFIGURACIÓN DE ACCESO REMOTO: ROUTERS MIKROTIK
	RDP: 1-) NOS DIRIGIMOS POR IP AL ROUTER Y NOS DESCARGAMOS LA HERRAMIENTA DE WINBOX.
	     2-) ACCEDEMOS A LA INTERFAZ DE CONFIGURACIÓN DEL ROUTER ESCRIBIENDO LAS CREDENCIALES
	     3-) ABRIMOS PUERTO 3389 (RDP) PARA UNA MÁQUINA EN CONCRETO QUE SE VA A USAR (DESTINATION PORT)
	     4-) ABRIMOS RDP Y ESCRIBIMOS LA IP.

	HP ZCentral: SE ACCEDE UTILIZANDO DOMINIO Y PUERTO AL QUE SE ASIGNA UN ORDENADOR EN WINBOX.            
		     
		     PASOS:
		     1-) INSTALAR ZCENTRAL SENDER EN EL ORDENADOR DE DESTINO DE LA CONEXIÓN.
		     2-) INSTALAR ZCENTRAL REMOTE BOOST EN EL ORDENADOR ORIGEN.
	             3-) ESCRIBIR DOMINIO + : + PUERTO. DESPUÉS ESCRIBIR CREDENCIALES DEL USUARIO QUE VA A ACCEDER AL PC REMOTO.
